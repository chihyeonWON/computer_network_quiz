16장 인터넷 자세히 살펴보기

1. URL의 어느 부분이 스킴입니까?
스킴은 URL을 읽기위한 형식을 지정하고 일반적으로 프로토콜 혹은 서비스를 참고

2. URL에서 스킴의 위치?
스킴은 문자열의 시작 부분 콜론 이중 슬래시 앞 공간을 차지

3. 인터넷에서 일반적으로 사용되는 네 가지 스킴?
http, https, ftp, ldap, file, mailto, pop

4. URL의 대상 디렉터리에서 파일 이름이 생략되어 있다면 대부분의 웹 서버에서 기본적으로 보내는 파일?
index.html

17장 HTTP, HTML 및 월드 와이드 웹

1. HTTP가 협상 단계를 지원하는 이유?
서버와 브라우저가 다른 세션 매개변수로 설정되어 있다면, 협상 단계를 통해 필요한 공통 설정에 동의할 수 있습니다.

2. 백엔드 데이터베이스를 포함한 웹 사이트가 있습니다. 사용자에게 해당 정보를 제공하기 위하여 서버사이드 스크립트를 사용하는 이유?
데이터베이스가 네트워크 연결의 서버 쪽에 있기 때문에 서버에서 완성된 코드를 조립하는 것이 더 효율적입니다.

3. 막 새로운 시스템을 설치해 잘 작동하고 있는데, 웹 사이트에 가서 PDF 파일을 클릭하면 열리지 않는다 어떻게 해결할까?
브라우저 혹은 운영 시스템에 따라서 알맞은 브라우저 플러그인을 설치하거나 PDF 리더를 PDF 파일 유형과 연관시켜 브라우저가 무엇을 해야 하는지 알려줘야 합니다.

4. RDF 트리플의 형식? Resource Description Framework 자원 기술 프레임워크
간단한 문장과 비슷, 주제, 술어, 객체(목적어) 부분으로 나뉩니다.

19장 암호화, 추적 및 개인 정보 보호

1. 세션 쿠키와 영구 쿠키의 차이점?
세션 쿠키는 특정 만료 기간에 할당되어 있지 않고 세션이 끝날 때까지 유지됩니다.
영구 쿠키는 주어진 만료 기간 동안 유지되고 사이트에 추후 방문시 사용자를 식별하기 위해 사용할 수 있습니다.

2. 웹 브라우저에서 추적 금지를 활성화했는데 광고 회사가 계속해서 나를 추적하는 것 같다. 잘못한 것은?
추적 금지는 자발적인 프로그램이며 많은 웹사이트가 여전히 지원하지 않는다. 따라서 잘못한 것은 없다.

20장 이메일

1. MIME은 무엇이며 어떤 용도로 사용됩니까?
Multipurpose Internet Mail Extension
MIME은 다목적 인터넷 우편 확장입니다. 이는 비ASCII 첨부 파일을 이메일 메시지로 인코딩합니다.

2. 이메일 메시지를 보낼 때 사용되는 프로토콜은 무엇입니까?
SMTP (Simple Mail Transfer Protocol)

Message Transfer (MT) 

3. 사용자의 메일박스에서 이메일 메시지를 검색하는 데 사용하는 프로토콜은 무엇입니까?
IMAP4, POP3

Message Acess (MA)

POP(Post Office Protocol)
IMAP(Internet Message Access Protocol)

4. 웹 메일의 가장 큰 문제점은 무엇입니까?
인터넷 병목 현상으로 인한 성능 저하입니다.

5. 웹 메일의 장점은 무엇입니까?
사용 및 구성이 쉽기에 비기술적인 사용자에게 좋은 선택입니다.
http를 사용하기 때문에 smtp, pop 그리고 imap에 닫혀 있는 방화벽을 통과할 수 있습니다.
인터넷에 접근해서 모든 컴퓨터에서 이메일을 확인할 수 있습니다.

http -> IMAP -> http
IMAP -> IMAP -> http
